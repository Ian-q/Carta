public API unchanged
Image crop extraction — for TEXT_WITH_IMAGES, instead of rendering the whole page and sending it to LLaVA, we extract each embedded image individually using PyMuPDF's page.get_image_rects() + fitz.Pixmap(doc, xref). Each crop gets its own LLaVA call and its own chunk. For the caption-fallback case (vector graphic, no get_images() results), we fall back to rendering the full page.

Returned chunks — PURE_TEXT pages return no image chunks (pipeline already has PyMuPDF text). TEXT_WITH_IMAGES returns one chunk per image crop + the PyMuPDF text is kept as a separate chunk by the existing pipeline. STRUCTURED_TEXT and FLATTENED return one chunk for the full page OCR result.

Does this structure make sense? One more section after this — config and the chunk output format — then we're done with the design.
