Design: Page Classification Decision Tree

Section 1 of 3 — The routing logic. All signals come from PyMuPDF metadata, zero model calls to classify.

# Stage 1: Gather signals (free, PyMuPDF only)
text = page.get_text().strip()
images = page.get_images() # embedded image list
has_tables = column_align_heuristic() # existing logic from classifier.py
has_caps = figure_caption_regex(text) # "Fig.", "Figure N", "see plot"

# Stage 2: Classify page
if len(text) > TEXT_MIN_CHARS: # e.g. 150 chars
  if has_tables:
    → STRUCTURED_TEXT # GLM-OCR
  elif images or has_caps:
    → TEXT_WITH_IMAGES # PyMuPDF text + LLaVA on crops
  else:
    → PURE_TEXT # PyMuPDF only — NO vision call
else: # little/no extractable text
    → FLATTENED # GLM-OCR → fallback LLaVA
PURE_TEXT
PyMuPDF chunk only
0 model calls
Most datasheet pages → fast
STRUCTURED_TEXT
GLM-OCR on full page render
1 model call
Register maps, spec tables
TEXT_WITH_IMAGES
PyMuPDF text chunk
+ LLaVA on each image crop
1 call per image
Timing diagrams, block schematics
FLATTENED
GLM-OCR → if yield < 50 chars
→ LLaVA full page
1–2 model calls
Scanned pages, image-only

TEXT_MIN_CHARS = 150 (configurable)  |  figure caption regex catches "fig.", "figure N", "see plot/chart/diagram"  |  LLaVA crops extracted via page.get_image_rects()
