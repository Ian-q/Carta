# Phase 999.1 Plan 02: Sidecar Stub Schema + Qdrant Lifecycle Payload

**JWT auth with refresh rotation using jose library** — Updated sidecar stub schema and Qdrant chunk payload to include lifecycle and generation fields, enabling generation-aware UUID management and stale/orphan tracking.

## Summary

Plan 999.1-02 extends the sidecar metadata schema and Qdrant payload structure with six new lifecycle fields. Every sidecar stub now carries `file_hash`, `hash_algorithm`, `generation`, `last_hash_check_at`, and `version_history` — metadata required to track when a file's content changes and manage version history. Every embedded chunk in Qdrant now includes `doc_generation`, `stale_as_of`, `superseded_at`, `orphaned_at`, `sidecar_id`, and `chunk_source_hash` — fields necessary for `lifecycle.py` to mark stale, detect orphans, and clean up expired chunks.

Additionally, two helper functions were added: `_point_id_versioned()` in `embed.py` for generation-aware UUID generation, and `collection_for_doc_type()` in `config.py` for routing protected document types to a dedicated `_notes` collection. The routing logic ensures that quirks, bug-notes, and helpful-notes are stored separately from regular documentation, enabling role-based recall strategies.

### Task 2: Payload Schema + _point_id_versioned + collection_for_doc_type (PAYLOAD-01)

**RED → GREEN → COMMIT**

1. Created failing tests in `carta/tests/test_embed.py`:
   - Tests for `_point_id_versioned()`: verifies UUID differs from legacy `_point_id` and differs per generation
   - Tests for upsert payload routing: checks that versioned ID is used when `doc_generation` present, legacy ID for backward compat
   - Tests for payload fields: verifies all six lifecycle fields present with correct initial values
   - Tests for defaults: verifies fields get safe defaults when absent from chunk dict
   - Tests for `collection_for_doc_type()`: maps protected types to `_notes`, session to `_session`, everything else to `_doc`

2. Implemented in `carta/embed/embed.py`:
   - Added `_point_id_versioned(slug, chunk_index, generation)` helper:
     - Generates UUID from `f"{slug}:{chunk_index}:g{generation}"` (note: `g` prefix for clarity)
     - Different generation values produce different UUIDs
     - Enables retries without point ID collisions
   - Updated `upsert_chunks()`:
     - Branches on `chunk.get("doc_generation") is not None`:
       - If present: calls `_point_id_versioned()` with generation value
       - If absent: falls back to `_point_id()` (backward compat with legacy chunks)
     - Adds six new fields to every point's payload:
       - `doc_generation`: int, default 1 if absent (safe default for legacy chunks)
       - `stale_as_of`: null (populated by `lifecycle.py::mark_sidecar_stale`)
       - `superseded_at`: null (populated by pipeline logic)
       - `orphaned_at`: null (populated by pipeline logic)
       - `sidecar_id`: str, default "" if absent (empty string for legacy chunks)
       - `chunk_source_hash`: str, default "" if absent
   - Preserved existing `_point_id()` function unchanged (used by legacy chunks)

3. Implemented in `carta/config.py`:
   - Added `collection_for_doc_type(cfg, doc_type)` helper function:
     - Protected types ("quirk", "bug-note", "helpful-note") → `{project}_notes` collection
     - Session type → `{project}_session` collection
     - All others (including unknown strings) → `{project}_doc` collection (safe default)
   - Used to route collections at Qdrant upsert time (caller responsibility; embed.py does not call this function yet)

4. All 10 tests pass. Commit: `38b3a0d`
