## Key Decisions

**1. Stale alert threshold and max_generations are configurable defaults**
- `embed.stale_alert_threshold` defaults to 0.30 (30%)
- `embed.max_generations` defaults to 2
- Fallbacks are in place; code continues safely if config missing

**2. Version history trimmed after append (not during)**
- Avoids pre-computing counts; entries appended, then list sliced to `[-max_gens:]`
- Ensures latest entry is always included

**3. Migration boundary guard is dual-layer**
- `pipeline.py` checks `sidecar.get("sidecar_id")` before calling `mark_sidecar_stale`
- `lifecycle.py::mark_sidecar_stale` also guards with `if not sidecar_id: return` (defense-in-depth)

## Technical Details

### Mtime Fast-Path Logic

```python
if not force and not needs_rehash(file_path, sidecar_data):
    return {"status": "skipped", "reason": "..."}
```

Short-circuits embed pipeline when file mtime matches sidecar record. Prevents redundant hash computation.

### Hash Comparison Logic

```python
current_hash = compute_file_hash(file_path)
if current_hash == old_hash and old_hash is not None:
    # Update mtime only
else:
    # Increment generation, mark stale, upsert new chunks
```

Guards against `old_hash is None` (never hashed before) so files on first pass are not falsely marked as unchanged.

### Generation and Version History

```python
new_generation = old_generation + 1
version_entry = {"hash": current_hash, "generation": new_generation, "indexed_at": now}
version_history.append(version_entry)
version_history = version_history[-max_gens:]
```

Tracks up to `max_gens` versions. Older entries are discarded, keeping disk footprint reasonable.

### Stale Alert

```python
threshold = cfg.get("embed", {}).get("stale_alert_threshold", 0.30)
alert_msg = check_stale_alert(stale_count, total_count, threshold)
if alert_msg:
    print(alert_msg, flush=True)
```

Alert fires after embed loop completes. Message is printed to stdout (not stderr) — safe for CLI context; not present in MCP server path.
