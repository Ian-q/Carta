## Objective

Implement four Qdrant-dependent lifecycle functions in `carta/embed/lifecycle.py` to support generation-aware chunk management. ## Key Design Decisions

1. **Pre-999.1 Migration Boundary:** Empty/None `sidecar_id` triggers silent no-op in both `mark_sidecar_stale` and `cleanup_expired_orphans`. This documents the transition: chunks embedded before Phase 999.1 are inert until re-embedded with Plan 02's updated schema.

2. **Frozenset for Protected Types:** `PROTECTED_DOC_TYPES` is a module-level frozenset constant. O(1) membership check, immutable, efficient.

3. **DatetimeRange API:** Uses `FieldCondition(key="orphaned_at", range=DatetimeRange(lt=cutoff))` not `DatetimeRange(key=..., lt=...)`. qdrant_client v1.7+ requires range as a nested structure within FieldCondition.

4. **Integer Percentage in Warning:** `check_stale_alert` converts float percentage to int for human readability in warning message.

## Integration Points

- **pipeline.py:** Will call `mark_sidecar_stale` and `cleanup_expired_orphans` during embed workflow
- **mcp/server.py:** Will call `is_protected_doc_type` to route chunks to correct collection
- **CLI:** Will display output from `check_stale_alert` during embed operations
