## User Constraints (from CONTEXT.md)

### Locked Decisions

1. **Hash Strategy:** SHA256 + CRLF→LF normalization for markdown; raw bytes for PDF. New sidecar fields: `file_hash` (hex), `hash_algorithm: sha256`.
2. **Change Detection:** mtime fast-path first; hash comparison only on mtime mismatch. New sidecar field: `last_hash_check_at` (ISO timestamp). If hash unchanged despite mtime change, update mtime only.
3. **Status Taxonomy:** `pending | embedded | stale | superseded | orphaned` — stored in both sidecar and Qdrant chunk payload.
4. **Staleness Alert Threshold:** 30% collection-wide (configurable as `embed.stale_alert_threshold`). Alert printed at end of `carta embed` run. Hook appends warning note for stale chunks at retrieval time.
5. **Search-Time Stale Behavior:** Return stale chunks with warning note appended (not filtered). Claude Code can autonomously trigger re-embed via MCP tool.
6. **Orphan Detection — All Three Triggers:** (1) source sidecar deleted, (2) `doc_generation` > `max_generations` behind current, (3) `file_hash` doesn't match any known version in sidecar history. Orphaned = marked immediately, deleted from Qdrant after `orphan_ttl_days` (default 30). Protected collections (`quirk`, `bug-note`, `helpful-note`) are exempt.
7. **Chunk Retention — Rolling Generation Window:** Keep last N generations per doc (default 2, configurable as `embed.max_generations`). Old chunks marked `superseded`, then `orphaned` after window exceeded. Protected doc_types skip rolling window.
8. **Qdrant Chunk Payload Schema — New Fields:** `doc_generation`, `stale_as_of`, `superseded_at`, `orphaned_at`, `sidecar_id`, `chunk_source_hash`.
9. **Doc Type / Memory Type Taxonomy:** `doc | datasheet | manual | quirk | bug-note | helpful-note | session`. Protected types use `{project}_notes` collection.
10. **Sidecar Schema — Updated Fields:** `file_hash`, `hash_algorithm`, `generation`, `last_hash_check_at`, `version_history` (last N hash entries).
11. **Claude Code Agent Autonomy via MCP:** `carta_embed(scope="stale"|"file"|"all")`. `scope="file"` requires `path` argument.

## Summary

This phase adds versioning and lifecycle infrastructure to Carta's embedding pipeline. The core work touches four areas: (1) sidecar schema expansion with hash and generation fields, (2) Qdrant chunk payload schema expansion with lifecycle metadata, (3) payload-only update operations in Qdrant (no re-embedding required), and (4) a new `carta_embed` scope parameter for MCP-driven targeted re-embedding.

All key technical questions were answered by inspecting the installed libraries directly (qdrant-client 1.17.1, mcp 1.26.0). Every planned operation is supported without library upgrades. The Qdrant client exposes `set_payload()` and `delete_payload()` that accept a `Filter` as the `points` argument — enabling bulk payload updates on matching chunks without touching vectors. `DatetimeRange` is available for `orphaned_at < now - 30d` queries. FastMCP's `@mcp.tool()` decorator handles `Literal['stale', 'file', 'all']` type hints natively, generating a proper `enum` JSON Schema.

**Primary recommendation:** Introduce `carta/embed/lifecycle.py` as the single home for orphan detection, staleness scanning, and generation cleanup. Keep `pipeline.py` orchestrating; have it call lifecycle functions at the right points in the embed run.

### Pattern 1: Payload-Only Update (No Re-embedding)

`client.set_payload()` accepts either a list of point IDs or a `Filter` as its `points` argument. Use the Filter form for bulk lifecycle state transitions.

```python
# Source: live inspection of qdrant-client 1.17.1
from qdrant_client.models import Filter, FieldCondition, MatchValue

# Mark all chunks from a sidecar as stale
client.set_payload(
    collection_name=coll_name,
    payload={"stale_as_of": datetime.now(timezone.utc).isoformat()},
    points=Filter(must=[
        FieldCondition(key="sidecar_id", match=MatchValue(value="docs/chip.embed-meta.yaml"))
    ]),
)
```

**Confirmed:** `set_payload` signature accepts `Filter` as `points` parameter. Returns `UpdateResult`. Does NOT require vectors to be re-computed.

### Pattern 2: DatetimeRange Filter for Orphan TTL Cleanup

`DatetimeRange` is available in qdrant-client 1.17.1 under `qdrant_client.models`. Fields: `lt`, `gt`, `gte`, `lte` — all accept `datetime | date | None`.

```python
from qdrant_client.models import Filter, FieldCondition, DatetimeRange
from datetime import datetime, timezone, timedelta

cutoff = datetime.now(timezone.utc) - timedelta(days=30)
expired_orphans_filter = Filter(must=[
    FieldCondition(key="orphaned_at", range=DatetimeRange(lt=cutoff))
])

# Scroll to collect IDs, then delete
points, _ = client.scroll(
    collection_name=coll_name,
    scroll_filter=expired_orphans_filter,
    with_payload=False,
    with_vectors=False,
    limit=1000,
)
ids = [p.id for p in points]
if ids:
    client.delete(collection_name=coll_name, points_selector=ids)
```

**Confirmed:** `DatetimeRange` serializes ISO 8601 timestamps correctly in filter JSON. `IsNullCondition` / `must_not` can be used to find non-null `orphaned_at`.

### Pattern 3: Scroll All Chunks for a Sidecar

`client.scroll()` accepts `scroll_filter: Filter`. Returns `(list[Record], next_offset)`. Use pagination for large collections.

```python
from qdrant_client.models import Filter, FieldCondition, MatchValue, PayloadSelectorInclude

all_points = []
offset = None
while True:
    batch, offset = client.scroll(
        collection_name=coll_name,
        scroll_filter=Filter(must=[
            FieldCondition(key="sidecar_id", match=MatchValue(value=sidecar_id))
        ]),
        limit=256,
        offset=offset,
        with_payload=True,
        with_vectors=False,
    )
    all_points.extend(batch)
    if offset is None:
        break
```

**Confirmed:** scroll returns `(records, next_offset_or_None)`. Pagination works by passing returned offset back in.

### Pattern 4: SHA256 File Hash with LF Normalization

```python
# stdlib only
import hashlib

def compute_file_hash(file_path: Path) -> str:
    """SHA256 with CRLF→LF normalization for text files; raw bytes for PDF."""
    raw = file_path.read_bytes()
    if file_path.suffix == ".md":
        raw = raw.replace(b"\r\n", b"\n")
    return hashlib.sha256(raw).hexdigest()
```

**Note:** `file_path.read_bytes()` reads the full file into memory. For files up to a few MB this is fine. For very large PDFs, a streaming approach exists but is not necessary for the doc sizes Carta targets.

### Pattern 5: mtime Fast-Path Before Hash

```python
def needs_rehash(file_path: Path, sidecar: dict) -> bool:
    """Return True if file should be re-hashed (mtime changed)."""
    current_mtime = os.path.getmtime(str(file_path))
    stored_mtime = sidecar.get("file_mtime")
    if stored_mtime is None:
        return True
    return current_mtime != stored_mtime
```

**Note:** `os.path.getmtime` returns a float (Unix timestamp). Already used in `pipeline.py:run_embed_file()` at line 201. No change needed to the mtime read pattern — just add hash comparison after mtime mismatch is detected.

### Pattern 6: FastMCP Scope Parameter

`Literal` type hints are natively handled by FastMCP 1.26.0. The decorator generates a correct `enum` JSON Schema automatically.

```python
from typing import Literal, Optional

@mcp_server.tool()
def carta_embed(
    scope: Literal["stale", "file", "all"] = "all",
    path: Optional[str] = None,
    force: bool = False,
) -> dict:
    """Embed documents into the project's vector store.

    Args:
        scope: 'all' embeds all pending files; 'stale' re-embeds changed files;
               'file' embeds a single file (requires path).
        path: Required when scope='file'. Absolute or repo-relative path.
        force: If True, re-embed even if file unchanged.
    """
```

**Confirmed schema output:**
```json
{
  "scope": {"type": "string", "enum": ["stale", "file", "all"], "default": "all"},
  "path":  {"anyOf": [{"type": "string"}, {"type": "null"}], "default": null}
}
```

### Anti-Patterns to Avoid

- **Storing ISO strings in Qdrant for DatetimeRange filters:** Store as ISO 8601 strings (`datetime.isoformat()`) in the payload; `DatetimeRange` handles parsing. Do NOT store Unix timestamps (float) for fields that will use `DatetimeRange` — that uses `Range` (float), not `DatetimeRange`.
- **Using `delete_payload` to null-out lifecycle fields:** Set the field to `null` explicitly via `set_payload` so `IsNullCondition` can find it. `delete_payload` removes the key entirely, which breaks `IsNullCondition` checks that rely on the key existing with a null value.
- **Blocking `run_embed` with synchronous Qdrant cleanup:** The lifecycle cleanup pass (orphan deletion) should be a separate callable invoked after the main embed loop, not woven inside the per-file thread executor.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Bulk payload updates | Loop over IDs calling set_payload once each | `client.set_payload(points=Filter(...))` | Filter-based bulk update is atomic and efficient |
| Datetime comparison in Qdrant | Python-side filter after scroll | `FieldCondition(key=..., range=DatetimeRange(...))` | Pushes filter to DB; avoids loading all records |
| Non-null field detection | Scroll all, filter in Python | `Filter(must_not=[IsNullCondition(is_null=PayloadField(key=...))])` | Qdrant-native, no Python post-filter |
| MCP enum validation | Manual if/elif on string scope | `Literal['stale', 'file', 'all']` type hint | FastMCP generates enum schema; MCP client validates before handler is called |
| Cross-platform hash stability | Custom line-ending detection | `raw.replace(b'\r\n', b'\n')` before `hashlib.sha256` | One-liner; covers the only relevant case |

## Common Pitfalls

### Pitfall 1: Overwriting `carta_embed` Instead of Extending

**What goes wrong:** Replacing the existing `carta_embed(path, force)` signature with the new `scope`-based one breaks any existing MCP client calls that pass `path` positionally.

**Why it happens:** The old tool takes `path: str` as a required first argument; the new design makes `path` optional.

**How to avoid:** Make `path: Optional[str] = None` in the new signature. Add a guard: if `scope == "file"` and `path is None`, return `{"error": "invalid_request", "detail": "path required when scope='file'"}`. The existing `force` parameter can remain.

**Warning signs:** Old MCP calls like `carta_embed("docs/chip.pdf")` would pass `path` as `scope` — add a runtime check for this case.

### Pitfall 2: `_point_id` Collision Across Generations

**What goes wrong:** `_point_id(slug, chunk_index)` is deterministic — re-embedding the same doc produces the same UUIDs. `client.upsert()` overwrites existing points by ID. Old-generation chunks with the same slug+chunk_index are silently replaced.

**Why it happens:** `_point_id` in `embed.py:72` uses `md5(f"{slug}:{chunk_index}")` — same inputs = same ID.

**How to avoid:** For versioned re-embeds, the new generation's chunks must use a generation-scoped ID: `_point_id(slug, chunk_index, generation)` — incorporate `generation` into the hash input so old and new generation chunks coexist in Qdrant until cleanup.

**Warning signs:** After re-embedding, `chunk_count` in Qdrant equals the new embed count but old-generation chunks are gone (not `superseded`) — they were silently overwritten.

### Pitfall 3: DatetimeRange Requires datetime Objects, Not Strings

**What goes wrong:** Passing an ISO string to `DatetimeRange(lt="2026-02-26T...")` raises a Pydantic validation error.

**Why it happens:** `DatetimeRange` fields are typed `Union[datetime, date, None]`, not `str`.

**How to avoid:** Always pass `datetime` objects: `datetime.now(timezone.utc) - timedelta(days=30)`. Use timezone-aware datetimes (UTC).

### Pitfall 4: Protected Collections Use Different `collection_name`

**What goes wrong:** Lifecycle code queries the `{project}_doc` collection for orphan cleanup but protected types (`quirk`, `bug-note`, `helpful-note`) live in `{project}_notes`. Cleanup logic misses them — or worse, the planner adds them to the rolling window inadvertently.

**Why it happens:** The existing `collection_name(cfg, "doc")` call in `upsert_chunks` hardcodes "doc" type. Protected types need `collection_name(cfg, "notes")`.

**How to avoid:** `generate_sidecar_stub()` and `upsert_chunks()` must both respect `doc_type` → collection mapping. A helper `collection_for_doc_type(cfg, doc_type)` in `config.py` should centralize this mapping.

### Pitfall 5: `version_history` in Sidecar Grows Unbounded

**What goes wrong:** Every re-embed appends to `version_history`. After 20+ re-embeds, the sidecar YAML becomes unwieldy.

**Why it happens:** No trim logic.

**How to avoid:** When writing `version_history`, keep only the last `max_generations` entries (same window as Qdrant retention). Trim during `_update_sidecar` when the new generation is written.

### Pitfall 6: `discover_pending_files` Only Finds `status: pending`

**What goes wrong:** The new `scope="stale"` MCP mode needs to find `status: stale` files, but `discover_pending_files()` filters to `status == "pending"` only (pipeline.py:54).

**Why it happens:** Existing discovery logic is status-specific.

**How to avoid:** Add `discover_stale_files(repo_root)` function (or make the status filter a parameter) that returns sidecars with `status: stale`.

import hashlib, uuid

def _point_id_versioned(slug: str, chunk_index: int, generation: int) -> str:

    raw = f"{slug}:{chunk_index}:g{generation}"

## Open Questions

1. **`_point_id` backward compatibility**
   - What we know: Existing embedded chunks in Qdrant have IDs computed without a generation suffix. If we switch to generation-scoped IDs, the first re-embed will create new points alongside old ones — it won't overwrite them. Old points will need to be marked `superseded` via `set_payload` by matching on `sidecar_id`.
   - What's unclear: Whether there are existing Qdrant collections with real data that the planner should treat as "generation 0" and migrate.
   - Recommendation: Plan a Wave 0 task that documents the migration strategy. For clean installs, no migration needed. For existing collections, the first re-embed should find old chunks (via `sidecar_id` scroll), mark them `superseded`, and proceed with new generation-scoped IDs.

2. **`sidecar_id` field not in existing chunks**
   - What we know: Currently embedded chunks do NOT have `sidecar_id` in their payload (the field is new).
   - What's unclear: The scroll-by-sidecar_id pattern will return zero results for all existing chunks until they are re-embedded.
   - Recommendation: The lifecycle cleanup pass should handle the case where `sidecar_id` is absent gracefully (don't error; treat as "pre-lifecycle" chunk). The orphan detection Trigger 1 (sidecar deleted) will need a fallback for chunks without `sidecar_id`.

3. **`{project}_notes` collection creation**
   - What we know: `ensure_collection()` creates a collection if absent. Protected types need `{project}_notes`.
   - What's unclear: Whether `upsert_chunks()` should be given a `collection_name` override or whether the caller always derives it from doc_type.
   - Recommendation: Add `collection_for_doc_type(cfg, doc_type) -> str` to `config.py` that maps protected types to `_notes` and all others to `_doc`. Pass result into `upsert_chunks()` directly.
