## What Was Built

**`carta/embed/pipeline.py` — Vision integration in `_embed_one_file()`:**

- After text chunks are upserted, a `if file_path.suffix == ".pdf":` block lazily imports and calls `extract_image_descriptions(file_path, cfg)`
- Image chunks built with `chunk_index = len(raw_chunks) + len(image_chunks_so_far)` — no UUID collision with text chunks
- `upsert_chunks()` called a second time for image chunks (same path as text chunks per D-10)
- `sidecar_updates` now includes `image_count` (images detected) and `image_chunks` (descriptions successfully embedded)
- `status` always set to `"embedded"` — D-14 specifies that `image_count>0, image_chunks=0` is a signal, not a failure

**`carta/embed/induct.py` — Sidecar schema extension:**

- `generate_sidecar_stub()` now includes `"image_count": None` and `"image_chunks": None` fields after `chunk_count`

**`carta/tests/test_pipeline.py` — 5 new integration tests in `TestVisionIntegration`:**

- `test_embed_one_file_pdf_calls_vision` — PDF path invokes vision, upsert called twice
- `test_embed_one_file_md_skips_vision` — Markdown path never touches vision module
- `test_image_chunk_index_offset` — First image chunk at index 5 when 5 text chunks exist
- `test_sidecar_stub_has_image_fields` — Stub dict has both keys defaulting to None
- `test_vision_unavailable_text_still_embeds` — Empty img_descs list → single upsert call, status=embedded, image_count=0, image_chunks=0
