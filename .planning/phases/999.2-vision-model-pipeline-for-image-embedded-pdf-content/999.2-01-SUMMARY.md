## What Was Built

`carta/embed/vision.py` — the core vision pipeline module:

- `VISION_PROMPT` — 200-char module-level constant for Ollama prompt
- `extract_image_descriptions(pdf_path, cfg) -> list[dict]` — public entry point
- `_convert_to_rgb_png(pix, fitz)` — CMYK-to-RGB conversion using `fitz.Pixmap(fitz.csRGB, pix)`
- `_scale_pixmap_if_needed(doc, page, xref, pix, fitz)` — caps any dimension at 2048px, aspect-ratio-preserving
- `_extract_embedded_images(doc, page, fitz)` — per-image fail-open extraction
- `_has_significant_vector_content(page)` — threshold >=3 drawings to skip trivial borders
- `_render_page_as_png(page, dpi=150)` — page-render fallback for vector-only pages
- `_call_vision_model(image_png_bytes, ollama_url, model, timeout)` — Ollama `/api/generate` call with `stream=False`

`carta/config.py` — added `"ollama_vision_model": "llava:latest"` to `DEFAULTS["embed"]`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] fitz must be imported at module level for test patching**
- **Found during:** Task 2 (GREEN) — 8 of 9 tests failed with `AttributeError: <module 'carta.embed.vision'> does not have the attribute 'fitz'`
- **Issue:** Plan specified lazy `import fitz` inside `extract_image_descriptions()` matching `parse.py` pattern. But `@patch("carta.embed.vision.fitz")` requires `fitz` to exist as a module-level name.
- **Fix:** Changed to `import fitz` at module top with `try/except ImportError: fitz = None` fallback, removing the lazy import inside the function.
- **Files modified:** carta/embed/vision.py
- **Commit:** d8a0429

**2. [Rule 1 - Bug] VISION_PROMPT was 204 chars, exceeding 200-char limit**
- **Found during:** Task 2 — `test_vision_prompt_constant` failed with `204 != <= 200`
- **Issue:** Plan said "196 chars" but actual string was 204 chars.
- **Fix:** Removed ", and" before "block labels" to reach exactly 200 chars.
- **Files modified:** carta/embed/vision.py
