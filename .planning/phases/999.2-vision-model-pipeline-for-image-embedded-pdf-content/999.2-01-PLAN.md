    Create `carta/embed/vision.py` with these components:

    **Module-level constant (per D-07):**
    ```python
    VISION_PROMPT = (
        "Extract all technical data from this image: data values, axis labels and units, "
        "register field names and bit widths, table cell contents, pin names, "
        "waveform annotations, and block labels. List each item."
    )
    ```
    This is 196 chars — within the <=200 target.

    **Internal helpers:**

    `_convert_to_rgb_png(pix) -> bytes`:
    - Check `pix.colorspace not in (fitz.csGRAY, fitz.csRGB)` — if not RGB/gray, convert: `pix = fitz.Pixmap(fitz.csRGB, pix)`
    - Return `pix.tobytes("png")`
    - Per Research Pitfall 1: never check `pix.n` alone, always check `pix.colorspace`

    `_scale_pixmap_if_needed(doc, page, xref, pix) -> fitz.Pixmap`:
    - If `pix.width > 2048 or pix.height > 2048`: compute `scale = 2048 / max(pix.width, pix.height)`, then re-extract at scaled resolution. Use `page.get_pixmap(clip=img_bbox, matrix=fitz.Matrix(scale, scale))` where `img_bbox` is derived from `page.get_image_rects(xref)[0]` if available, otherwise skip scaling and use original pix.
    - Per D-05: cap any dimension at 2048px, aspect-ratio-preserving.
    - If scaling mechanism unavailable (API check), fall back to using original pix (fail-open).

    `_extract_embedded_images(doc, page) -> list[bytes]`:
    - Call `page.get_images(full=True)` to get xref tuples
    - For each xref: `pix = fitz.Pixmap(doc, xref)` -> scale if needed -> convert to RGB PNG
    - Wrap each extraction in try/except, print warning to stderr on failure, continue (per D-11)
    - Return list of PNG bytes

    `_has_significant_vector_content(page) -> bool`:
    - Return `len(page.get_drawings()) >= 3`
    - Per Research Pitfall 6: threshold of 3 to skip trivial borders

    `_render_page_as_png(page, dpi=150) -> bytes`:
    - `pix = page.get_pixmap(dpi=dpi)` then `pix.tobytes("png")`
    - Per D-05: 150 DPI for fallback

    `_call_vision_model(image_png_bytes, ollama_url, model, timeout=60) -> str`:
    - `base64.b64encode(image_png_bytes).decode("utf-8")`
    - `requests.post(f"{ollama_url}/api/generate", json={"model": model, "prompt": VISION_PROMPT, "images": [b64], "stream": False}, timeout=timeout)`
    - On non-200 status: raise RuntimeError with status and first 200 chars of body
    - Return `resp.json()["response"].strip()`

    **Public entry point (per D-04, D-05, D-08, D-09, D-11):**

    `extract_image_descriptions(pdf_path: Path, cfg: dict) -> list[dict]`:
    - Lazy `import fitz` inside function (matches parse.py pattern)
    - Read `ollama_url` from `cfg["embed"]["ollama_url"]`
    - Read `model` from `cfg["embed"].get("ollama_vision_model", "llava:latest")`
    - Open PDF: `doc = fitz.open(str(pdf_path))`
    - For each page:
      - Get embedded images via `_extract_embedded_images(doc, page)`
      - For each image PNG: call `_call_vision_model()`, wrap in try/except (fail-open per D-11), log warning to stderr on failure
      - If no embedded images AND `_has_significant_vector_content(page)`: render page via `_render_page_as_png()`, call vision model on rendered PNG
      - Each successful result appended as: `{"page_num": page_num, "image_index": idx, "doc_type": "image_description", "text": description}`
    - Close doc, return results list
    - If any exception during PDF open, print warning to stderr and return []

    **Style rules:**
    - `import fitz` inside `extract_image_descriptions()` only (lazy)
    - `import base64`, `import requests`, `import sys` at module top
    - Type hints on all public functions
    - Warnings via `print(f"Warning: ...", file=sys.stderr, flush=True)`
    - 4-space indent, PEP 8
