    `_extract_query(prompt: str, cfg: dict) -> str` (per D-06):
    - If `len(prompt) <= 500`: return prompt
    - Else: try calling Ollama to compress (same model as judge, same timeout):
      ```python
      ollama_url = cfg["embed"]["ollama_url"]

          "messages": [
              {"role": "system", "content": "Extract a concise 1-2 sentence search query from this text."},
              {"role": "user", "content": prompt[:1000]},

      return resp.json()["message"]["content"].strip()
      ```

      Documentation candidates:
      {excerpts}

      Are any of these relevant?

    - System message (per D-15): `"You decide if documentation is relevant to a coding prompt. Answer only 'yes' or 'no'."`
    - POST to `{ollama_url}/api/chat` with `{"model": model, "messages": [...], "stream": False}`, timeout=4
    - Parse (per D-17): `response.json()["message"]["content"].strip().lower().startswith("yes")`
    - Return True if starts with "yes", False otherwise

    `_judge_with_timeout(prompt: str, hits: list[dict], cfg: dict, timeout_s: int) -> bool` (per HOOK-05, D-09):
    - Use `concurrent.futures.ThreadPoolExecutor(max_workers=1)`
    - Submit `_call_ollama_judge(prompt, hits, cfg)`
    - `future.result(timeout=timeout_s)` — on TimeoutError return False (fail-open)
    - On any other exception return False

    - Use `sys.__stdout__.write(...)` to bypass any stdout redirection if used
